C - Triangular Relationship
基本的には気づきゲーであり、
Kが奇数ならa%K, b%K, c%Kが全て0⇔条件を満たす
Kが偶数ならa%K, b%K, c%Kが全て0または全てK/2⇔条件を満たす
である。

気づくまでの過程として、以下全部 mod K として、問題の設定の式
a + b = 0
b + c = 0
c + a = 0
を立て、cを消してみると2a = 0が出てくる。
よって 2a = 0 は必要条件であり、同じようにして2b = 0, 2c = 0も必要条件になるのがわかる。
ここでKが奇数の時を考えると、a=b=c=0が必要条件ということになる上、これは十分条件でもあることが確認できる。
Kが偶数の時を考えると、a,b,cがK/2か0であるのが必要条件ということになる。a,bのみK/2だとb+c=0にならないのでダメ、aのみK/2だとa+b=0にならないのでダメ、ということを考えると、許されるのはa=b=c=0とa=b=c=K/2だけということがわかる。
という感じで気づくことができる。

D - All Your Paths are Different Lengths
構築ゲー。かなり難しかった。

とりあえずいろいろ実験すると下に気づく。
L=2^rであるならばこの問題は簡単。
というのもこれは2進数をやればよくて、
「頂点1~r+1を作る(N=r+1になる)」
「頂点iとi+1の間に辺0と2^(i-1)を張る」
だけでよい。
L <= 10^6 < 2^20(=1048576) なので r < 20となり、 N=r+1 <= 20 になるため問の指定も満たす。

じゃあこれに辺を追加で張ることでどうにかしたいなという気持ちになる。
rを 2^r <= L であるような最大のrであるとする。（Nの制約に関しては上と同様に守られるのがわかる。）

ここで一般の点sからtに張ることを考えるとすごく混乱してしまうので、始点=1もしくは終点=Nを固定した上で張りたい。
重み0の辺を、始点=1を固定して適当なtに張ると重みのでかい辺と0とが合わせて2^(N-t)個できるが、終点=Nを固定して適当なsから張ると重み 0 ~ 2^(s-1)-1 までの2^(s-1)個の辺ができるのがわかる。sから張ったほうが重みが連続していて取り回しがよさそうである。というか、重みXの辺を張ることにすると、重みX ~ X+2^(s-1)-1までの2^(s-1)個の辺ができる。よさそう。

ということで「終点=Nを固定して適当なsから張る」ことを考えるのだが、これらの辺により新たにできるパスはそれぞれ干渉しあわないこともわかる。よってこの方針でガンガン張っていけばよさそう。

個数的に考えると、L-2^rの2進数表現で(1-indexed)s bit目が立っていればs->Nを作ればいいということになるだろう。
各辺の重みXは、最終的に2^r<=Y<Lを覆えてさえいれば問題ない。
よってこれをなんとかして実装すればよい。解説では、頂点sの候補を逆順に見ていき、個数的に張る必要があると判断されたら重みとしては2^r<=Y<Lの小さい方から順に埋めていくような重みを設定する、という方法で張っている。
提出では何をやっているのかもはやよくわからない。
